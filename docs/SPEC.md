# DINODEX — Product & Technical Specification

> A Pokédex-style web encyclopedia for dinosaurs with anime-styled art generated by Gemini.
> MVP: 30 curated dinosaurs, 3 evolution stages each, fun and engaging UI.

---

## 1. Product Overview

### 1.1 Vision
Dinodex is a web app that presents dinosaur information in the style of a Pokédex — each dinosaur has a unique number (#001–#030), anime-styled artwork for 3 growth stages (Hatchling → Juvenile → Adult), and detailed stats presented as visual bars. The experience should feel fun, playful, and addictive to browse — like a collectible encyclopedia.

### 1.2 Target Audience
- Dinosaur enthusiasts of all ages
- Parents/kids looking for educational-but-fun dino content
- Pokémon fans who'd enjoy the crossover format
- Casual browsers who stumble on it and get hooked

### 1.3 MVP Scope
- 30 curated popular dinosaurs spanning all 3 geological eras
- 4 core features: Browse grid, Detail page, Search/Filter, Evolution stages
- Pre-generated anime art via Gemini (90 images total)
- Responsive web app (mobile-first, desktop-friendly)
- Static deployment (no backend, no auth, no database)

## 3. Feature Specifications

### 3.1 Home Screen — Dino Grid

**Layout:**
- Mobile: 2-column card grid, scrollable
- Tablet: 3-column grid
- Desktop: 4-column grid with max-width container (1200px)

**Card Component (`DinoCard`):**
- Fixed aspect ratio card with colored border matching dino type
- Top: Pokédex-style number in monospace (#001), small, top-left
- Center: Anime art image (adult stage by default), contained within card
- Bottom: Dino name (bold), diet icon + label below
- Border color mapping:
  - Carnivore: Red (#DC2626)
  - Herbivore: Green (#16A34A)
  - Omnivore: Amber (#D97706)
  - Piscivore: Blue (#2563EB)
- Hover: Card lifts (translateY -4px), shadow grows, subtle border glow
- Click: Navigates to /dino/[id] with page transition

**Search Bar:**
- Sticky at top of grid area (below header)
- Instant client-side filter on name
- Placeholder: "Search dinosaurs..."
- Clear button (×) when text is present

**Filter Chips:**
- Horizontal scrollable row below search
- Groups: Era (All, Triassic, Jurassic, Cretaceous) | Diet (All, Carnivore, Herbivore, Omnivore)
- Active chip: filled with accent color, white text
- Inactive chip: outlined, muted text
- Multiple filters combine (AND logic): selecting "Jurassic" + "Carnivore" shows only Jurassic carnivores
- "All" chip in each group resets that group's filter

**Header:**
- App logo/title: "DINODEX" in display font with dino emoji or small icon
- Subtitle: "Dinosaur Encyclopedia" in smaller muted text
- Fixed/sticky on scroll

**Empty State:**
- When filters produce 0 results: friendly message with dino illustration
- "No dinosaurs found. Try different filters!"

### 3.2 Dino Detail Page

**Layout (mobile-first):**
- Full-width art hero section at top (16:9 aspect ratio)
- Content below in single column
- Desktop: art on left (40%), content on right (60%), or stacked with larger art

**Hero Section:**
- Large anime art image for the currently selected evolution stage
- Background gradient matching the era color
- Pokédex number overlay: large "#018" in semi-transparent white, top-left
- Name below art in display font
- Meaning in italic: "Tyrant Lizard King"
- Pronunciation: "tie-RAN-oh-SORE-us REX" in mono font

**Evolution Stage Selector:**
- 3 pill buttons in a row: "🥚 Hatchling" | "⚡ Juvenile" | "🔥 Adult"
- Active stage: filled with stage color, elevated
- Hatchling color: Green (#16A34A)
- Juvenile color: Amber (#D97706)
- Adult color: Red (#DC2626)
- Clicking a stage:
  1. Art crossfades to the new stage's image (300ms ease)
  2. Stat bars animate to new values (500ms spring)
  3. Description text fades and updates
  4. Background gradient subtly shifts

**Stats Panel:**
- Section heading: "Stats" with stage indicator
- 6 stat rows, each with:
  - Label (left): "Height", "Length", "Weight", "Speed", "Danger", "Defense"
  - Visual bar (center): filled proportional to max in dataset
  - Value (right): actual number with unit
- Bars animate on page load and on stage change
- Bar colors: can match a single accent or vary by stat
- Stats are stage-specific — they change when you switch stages

**Info Section:**
- Card/panel below stats with:
  - Era + period: "Late Cretaceous (68–66 Mya)"
  - Region: "North America"
  - Type: "Theropod"
  - Diet: "Carnivore" with icon
  - Locomotion: "Biped"
  - Discovered by: "Henry Fairfield Osborn, 1905"
- Fun fact: 2–3 sentence description, changes per stage

**Related Dinosaurs:**
- Horizontal scroll row at the bottom
- 3–5 mini cards of related dinos (same era or same family)
- Clicking navigates to that dino's detail page

**Back Navigation:**
- "← Back to Dinodex" link/button at top
- Or: browser back button works (Next.js routing)

### 3.3 Search & Filter

**Search:**
- Client-side instant filter using string matching on `name` field
- Case-insensitive
- Debounced input (150ms)

**Filters:**
- Era: Triassic | Jurassic | Cretaceous (single-select, or "All")
- Diet: Carnivore | Herbivore | Omnivore (single-select, or "All")
- Both filters applied as AND conditions
- URL query params reflect filter state: `/?era=jurassic&diet=carnivore`
- Filters persist across navigation (via URL state)

### 3.4 Evolution Stages

This is the hero feature. Each of the 30 dinosaurs has 3 stages with distinct:

**Stage 1: Hatchling 🥚**
- Art style: Chibi/kawaii anime, large head, big round eyes, soft lines
- Stats: ~10-20% of adult values
- Diet: Simplified (insects for carnivores, soft plants for herbivores)
- Description: Focuses on birth, early behavior, vulnerability
- Card accent: Soft green (#BBF7D0)

**Stage 2: Juvenile ⚡**
- Art style: Shonen manga, dynamic poses, motion lines, developing features
- Stats: ~40-60% of adult values
- Diet: Transitioning to adult diet
- Description: Focuses on growth, learning, pack dynamics
- Card accent: Warm amber (#FDE68A)

**Stage 3: Adult 🔥**
- Art style: Epic detailed anime, dramatic lighting, aura effects, full features
- Stats: 100% values
- Diet: Full adult diet
- Description: Focuses on apex behavior, hunting/defense strategies
- Card accent: Bold red (#FECACA)

**Stage Transition Animation:**
1. User taps stage button
2. Current art fades out (200ms)
3. Brief sparkle/particle burst (300ms) — CSS only
4. New art fades in (300ms)
5. Stat bars animate to new values with spring easing (500ms)
6. Description text crossfades (200ms)

## 4. Technical Architecture

### 4.1 Tech Stack

| Layer | Technology | Reason |
|-------|-----------|--------|
| Framework | Next.js 15 (App Router) | SSG for 30 static pages, image optimization, great DX |
| Language | TypeScript | Type safety for dino data model |
| Styling | Tailwind CSS v4 | Rapid prototyping, custom design tokens |
| Animation | Framer Motion | Spring physics, layout animations, page transitions |
| Data | Static JSON file | 30 entries, no DB overhead, version-controlled |
| Art | Gemini API (pre-generated) | 90 WebP images stored in /public |
| Deployment | Vercel | Free tier, auto-deploy, edge CDN |
| Fonts | Google Fonts | M PLUS Rounded 1c (display), Noto Sans (body) |

### 4.3 Data Flow

```
dinos.json (static)
    │
    ├──> Home Page (SSG at build time)
    │       │
    │       └──> Client: useFilteredDinos() hook
    │               ├── search query state
    │               ├── era filter state
    │               └── diet filter state
    │
    └──> Detail Page (SSG, generateStaticParams)
            │
            └──> Client: useState for active stage
                    ├── stage changes art image
                    ├── stage changes stat values
                    └── stage changes description
```

### 4.4 Image Strategy

- All 90 images pre-generated via Gemini before build
- Stored as WebP in `/public/dinos/[id]/[stage].webp`
- Card thumbnails: 512×512px
- Detail hero: 1024×1024px
- Next.js `<Image>` component handles optimization, lazy loading, blur placeholder
- Generate blur data URLs at build time for instant placeholders

### 4.5 Performance Targets

- Lighthouse: 95+ on all metrics
- First Contentful Paint: <1s
- Time to Interactive: <2s
- Total page weight (home): <500KB
- All 30 dino pages statically generated at build time

## 6. Non-Functional Requirements

### 6.1 Accessibility
- All images have descriptive alt text
- Color is not the only indicator (icons accompany diet colors)
- Keyboard navigable: Tab through cards, Enter to open
- Focus visible states on all interactive elements
- Minimum contrast ratio 4.5:1 for all text

### 6.2 SEO
- Each dino page has unique `<title>` and `<meta description>`
- Open Graph images (auto-generated or static)
- Structured data (JSON-LD for each dino)
- Semantic HTML throughout

### 6.3 Responsive Design
- Mobile-first: 320px minimum width
- Breakpoints: sm(640), md(768), lg(1024), xl(1280)
- Touch targets: minimum 44×44px
- No horizontal scroll (except filter chips and related dinos)
